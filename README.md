# portal-baccarat-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面彼此独立，不依赖于特定域名或网站环境，适合作为静态页面集合进行托管与分发。

## 项目简介

portal-baccarat-pages 是一个纯前端页面归档仓库。仓库内包含若干自行封装的 HTML 文件，每个文件均可独立运行，无需后端支持。该仓库主要服务于页面资源的存档、备份以及通过 GitHub Pages 等方式进行静态发布。

## 目录说明

```
portal-baccarat-pages/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
├── index.html      # 可选的索引页，用于导航或展示页面列表
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件代表一个独立的 HTML 页面。页面之间无耦合，可单独访问。
- **assets/**：存放页面共用或专用的静态资源。资源文件按需组织，方便维护。
- **index.html**：入口页面，可展示当前仓库内的页面列表或提供跳转链接（非必须）。

## 页面归档说明

- 所有页面均为静态 HTML，支持现代浏览器。
- 页面内不包含任何外部跟踪、广告或动态服务端请求。
- 归档的页面可能包含示例数据、占位内容或演示逻辑，仅供存档参考。
- 部分页面可能引用公共资源（如字体、图标库），但均通过相对路径或 CDN 引用，不绑定特定域名。

## 维护说明

- 本仓库由维护者不定期更新，新增页面或修正内容。
- 欢迎通过 Issue 提交问题或建议，但请注意本仓库不针对任何具体网站或服务。
- 如需本地使用，可直接克隆仓库并通过本地服务器（如 `python -m http.server`）访问。
- 所有页面内容仅供存档与展示，不构成任何形式的商业或运营承诺。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/portal-baccarat-pages.git
   ```
2. 使用任意 HTTP 服务器打开 `index.html` 或直接访问 `pages/` 下的独立页面。
3. 也可通过 GitHub Pages 等功能直接发布到公网（需自行配置）。
